TypeScript - Get started

Posted on 2021-09-13 In Typescript

在去年轉職成為前端工程師後就開始使用了 Angular 這個框架，這個框架與其他兩個主流框架最大的不同點在於他預設就是使用 Typescript 作為開發語言，雖然在進公司前完全沒有聽過這個是什麼東西，不過隨著時間的推移漸漸發現 Typescript 的魅力，可以很好的避免很多 Javascript 的地雷，尤其是對於 Javascript 這個弱型別的語言來說，常常會出一些非常奇怪的錯誤，所以在編寫程式的時候要時常注意有沒有冒犯到 Javascript 的逆鱗導致錯誤，不過有了 Typescript 後就可以有效的避免一些奇怪的錯誤，它賦予了 Javascript 型別並可以對變量的內容進行檢查，從而使變量的內容更加明確與可追蹤，不過我也是使用了這麼久也沒有仔細的看 Typescript 的官方文檔，趁著最近專案比較輕鬆來從投讀一次官分文檔並做個紀錄，希望可幫助到剛接觸到 Typescript 的夥伴。

Get started

要在本機中使用到 Typescript 的環境首先需要在你的本機中安裝 NodeJS，這邊就不講解該怎麼安裝 node 了，安裝完成後可以打開終端機再隨意的地方創建一個新的檔案夾作為 Typescript 專案的存放位置

接著在建立好的檔案中將 npm 進行初始化，這樣才可以在專案中安裝 Typescript

建立完專案的檔案後，需要在檔案中安裝 Typescript，可以到 Typescript 官網 中下載也可以使用 npm 進行安裝

最後將專案的 typescript 進行初始化

Defining Types

再開發 Javascript 時可以使用很多種不同的設計模式，不過有些設計模式會讓開發者或未來的維護者很難知道這些變數的型別（ dynamic programming 設計模式 ），為了面免這種情況 Typescript 支持 Javascript 的擴展，它提供了一種告訴你這些變數是什麼型別的方法，比如可創建一個 object 並告訴他應該要包含一個 name: string 與 id: number 的 property

先創建一個 interface 定義好所有 property 的型別，之後將 object 的型別定義為這個 interface，這樣一來所創建的 object 就必須符合 interface 所規定的 property 類型，如果創建出來的 object 結構不符合 interface 定義的話則會出錯

除了創建變量可以定義型別之外，也可以對 Function 的參數定義類型，可以定義傳入 function 的參數的型別與定義傳出 function 的數據的型別

對於定義的型別類型 Javascript 有一組基本的類型可以使用，他們分別是 boolean, bigint, number, string, symbol 和 undefined，這些可以在 Typescript 中使用，不過除了這些型別之外 Typescript 有多幾種型別可以使用，比如 any, unknown, never 和 void 這四種，他們詳細是定義什麼狀況在後面會詳細的一一講解，

Composing Types

除了上面介紹的定義變量的型別之外，還可以使用組合簡單的型別而創建必較複雜的型別，比如說可以使用 Unions 或 Generics 這兩種方式建立複雜的類型定義

Unions

可以使用 Unions 將一個變數定義為一種或多種型別，比如你可以將一個 boolean 類型描述為 true 或 false

也可以將使用 Unions 將類型定義為是 string 或 number 的集合

Generics

Generics 可以為型別提供變量，一個常見的例子就是陣列，沒有設定 Generics 的陣列可以填如任何型別的數據，但可以使用 Generics 指定只有哪一種型別的數據可以放進此陣列中

如果指定陣列只能存放 string 類型的資料的話，就不能將非 string 的數據存放進此陣列中。

Structural Type System

對於 Typescript 的核心原則之一是檢查值的形狀而非值的內容，這種行為稱為 duck typing 或 structural typing，所以在注重結構的系統中，如果兩個 object 具有一樣的形狀那麼他就會被認定為同一類型，舉個例子

雖然 point3 與 rect 他們的內容都不是百分百的符合 Point 所定義的結構，但是由於他們都具有 property x 與 y 所以以形狀來說與 Point 是一樣的，所以 Typescript 會同意讓它通過編譯，但 color 的形狀卻完全不一樣，所以會報錯。

結論

本章簡單介紹該如何設置 Typescript 並使用它，也介紹了 Typescript 可以為使用者帶來什麼樣的便利與如何提高程式的安全性，無論是已開發者的立場還是維護者的立場都會希望能夠了解變量的型態，有時候甚是不需要去看他的說明光看他的型別就大概知道這個變量是做什麼用的了，所以了解 Typescript 是對開發大型專案來說是滿重要的，可以有效的避免錯誤的發生與增加可閱讀性。

後面也大概的講解了一些 Typescript 的特色，如何對變量定義型別與利用 Union 或 Generics 建立更複雜的型別等等，這邊只是大概的介紹一下概念，詳細的內容還是會在之後的章節做一個比較深入的講解。
